# visual-style-cloner

视觉风格克隆生图 Skill。

这个 Skill 用于把参考图的视觉风格逆向成可复用的图像生成提示词，并把新的中文标题、画面结构和总结文案重排成适合 ChatGPT 生图工具使用的图片 Prompt。它会在生成图片后继续询问是否满意；如果用户满意，还可以继续生成一版 3–5 秒的动态视频提示词，用于即梦、可灵等图生视频工具。

## 适用场景

- 参考图风格复刻
- 小红书封面、知识卡片、图文海报
- 把一段内容整理成单画面或 2–3 个独立画面
- 用 ChatGPT 生图工具直接生成图片
- 为已生成图片补充 3–5 秒动态提示词

## 核心流程

Skill 会按顺序推进，并在关键节点等待用户确认：

1. 检查是否有参考图，并逆向分析视觉风格。
2. 展示风格分析结果，提炼图片文案和中间画面结构，等待确认。
3. 询问图片比例：横版 16:9、竖版 3:4 或方版 1:1。
4. 生成最终图片 Prompt，并调用 ChatGPT 生图工具生成图片。
5. 图片生成后询问是否满意；满意后可继续生成 3–5 秒动态视频提示词。

## 视觉逆向内容

当用户提供参考图时，Skill 会分析并展示：

- 媒介和质感
- 线条和边缘特征
- 色彩和光影逻辑
- 角色造型语言
- 字体和文字层级
- 构图、网格、留白率和间距
- 负向特征，也就是该风格明确不应该是什么
- 可复用的英文风格关键词

## 中间画面结构逻辑

中间区域的画面数量只由文案的信息结构决定：

- 一个核心情境、动作、情绪或观点：使用一个完整场景。
- 两个步骤、并列点、对象或前后对比：拆成两个独立画面。
- 三个步骤、并列点或流程：拆成三个独立画面。
- 超过三个信息单元：先合并成 2–3 个 coherent units；如果会损失关键信息，则先询问用户。

参考图只决定视觉风格，不决定画面数量。图片比例只决定这些画面如何排列。

## 动效提示词逻辑

如果中间区域是单画面，动态提示词会要求所有动作在同一场景内连续完成。

如果中间区域包含 2–3 个独立画面，动态提示词会强制要求画面按阅读顺序一个一个出现：

1. 第一个画面淡入并轻微放大。
2. 画面内部发生小幅动态。
3. 短暂停留，保证可读。
4. 慢慢淡出。
5. 第二个画面再出现，依次类推。

同时约束标题、底部总结、背景和整体画框保持固定，不允许文字变形、画面漂移或多个画面同时抢占视觉中心。

## 包含内容

```text
visual-style-cloner/
  SKILL.md
  agents/
    openai.yaml
  references/
    prompt-template.md
```

- `SKILL.md`：Skill 主体工作流。
- `agents/openai.yaml`：ChatGPT / Codex 侧展示名、简介和默认调用提示。
- `references/prompt-template.md`：图片 Prompt 模板和动态提示词模板。

## 安装方式

### ChatGPT / Codex 桌面端

把 `visual-style-cloner` 目录复制到本机 skills 目录：

```bash
mkdir -p ~/.agents/skills
cp -R visual-style-cloner ~/.agents/skills/
```

然后新建任务，输入 `$`，搜索并选择：

```text
visual-style-cloner
```

也可以直接描述需求，例如：

```text
用 visual-style-cloner 逆向这张参考图的风格，并帮我生成一张新的中文图文海报。
```

### ChatGPT 网页端

如果你已经在 ChatGPT 网页端安装了这个 Skill，可以通过 Skill 入口调用；如果网页端没有安装入口，可以使用本仓库中的 Skill 源码作为本地 Codex / ChatGPT Worker 版本。

## 使用建议

- 上传参考图时，尽量选择风格清晰、元素不太拥挤的图片。
- 如果想高度贴近参考风格，生成图片时应同时提供参考图和最终 Prompt。
- 新内容的文案要短，尤其是标题和每个中间画面的可见文字。
- 如果中间信息超过三个点，建议先合并或拆成多张图。
